# 📊 SQL E-commerce Sales & Revenue Analysis (SQL Project)

## 🔹 Objective
This project analyzes E-commerce sales data to identify sales & revenue generated by customers using SQL.  
The goal is to extract meaningful business insights such as top customers, top categories, monthly revenue trends, and growth patterns.
This analysis helps businesses identify total revenue generated, total orders, detecting customer retention rate, top categories per month and make data-driven decisions for performance improvement.

---

## 🔹 Dataset Description

The project uses four tables:

- **customers**  
  Contains customer details.

- **orders**  
  Contains order details such as id, bought by, date and amount.

- **products**  
  Contains product details.

- **order_items**  
  Contains detail of ordered items with their quantities.
---

## 🔹 Key Analysis Performed

- Total revenue  
- Total orders  
- Total customers  
- Revenue by Category 
- Monthly revenue trend
- Average Order Value 
- Repeat customers (>= 2 orders) 
- Customers who ordered only once
- Retention Rate  
- Top category per month
- Running Total Revenue(Monthly)

---

## 🔹 SQL Concepts Used

- Joins  
- Aggregations (`SUM`, `COUNT`, `AVG`)  
- Window Functions (`ROW_NUMBER`, `SUM`)  
- Common Table Expressions (CTEs)  
- Conditional Logic (`CASE`)   

---

## 🔹 Key Insights
- Electronics dominate revenue showing strong dependency on high-ticket products
- There is a revenue spike in November
- Majority customers are repeat buyers (~60%), indicating decent retention
- Categories like Home and Accessories underperform
- Revenue trend help identify monthly performance
